# Contributing Guide

Thank you for your interest in contributing to MOLE! This document provides guidelines for contributing to the project.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the repository.
2. **Create a Branch**: Create a branch for your feature or bugfix.
3. **Make Changes**: Implement your changes, following the coding standards.
4. **Test Your Changes**: Ensure your changes don't break existing functionality.
5. **Submit a Pull Request**: Submit your changes for review.

## Reporting Issues

If you find a bug or have a suggestion for improvement, please open an issue on the GitHub repository.

## Getting Help

If you need help with using MOLE or contributing to it, please reach out to the maintainers or open a discussion on GitHub.
